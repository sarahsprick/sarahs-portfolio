DIY Spotify Wrapped 2022    

December 2022

  • Jupyter Notebook
  • GitHub Repository
  • Tableau Dashboard

As a long-time Spotify Premium subscriber and an avid listener of the "Made For You" playlists (created by Spotify to "help you discover more of what you love"), I have always been curious about the metrics used to determine what I may like. I have also been curious about whether my Spotify Wrapped provided at the end of each year is accurate.

When this year's Spotify Wrapped was released, I did a little research on what personal data was available to me from Spotify. Turns out, they give you access to quite a bit. To dig into this idea, I used data both provided by Spotify and downloaded by Spotify’s API about my account to see what songs/artists dominate my streaming history, when I listen the most, whether there are any common audio features, and more!

Data

Streaming History

The first section uses my streaming history using my account data found here. The turnaround time for this request took around ~5 days. I received a .zip folder with multiple .json files about my playlists, search queries, payment, followers/following accounts, stream history, etc. from the last 12 months. For this project, I am going to be focusing on the latter (streaming history).

I recieved 3 .json files of streaming history because there is a max of 10,000 rows per file. Used a for loop to append each file to one large DataFrame. 


list = [sm0, sm1, sm2]

streams = pd.DataFrame()

for df in list:
streams = streams.append(df, ignore_index=True)

Looking at Spotify's documentation to Understanding My Data, the DataFrame contains the following:

         A list of items (e.g. songs, videos, and podcasts) listened to or watched in the past year, including:
              ⬥ Date and time of when the stream ended in UTC format (Coordinated Universal Time zone).
              ⬥ Name of "creator" for each stream (e.g. the artist name if a music track).
              ⬥ Name of items listened to or watched (e.g. title of music track or name of video).
              ⬥ “msPlayed”- Stands for how many mili-seconds the track was listened to.

Converted the endTime column to datetime format, then created new columns called day, time, and day_week using this information. Created a new column called minutes_played which converts msPlayed from milliseconds to minutes. Renamed columns to a more readable format. 


streams['endTime'] = pd.to_datetime(streams['endTime'])
streams['day'] = [d.date() for d in streams['endTime']]
streams['time'] = [d.time() for d in streams['endTime']]
streams['day_week'] = pd.DatetimeIndex(streams['endTime']).day_name()

streams['minutes_played'] = streams['msPlayed'].divide(60000)
streams = streams.drop(['msPlayed'], axis=1)

streams = streams.rename(columns={'endTime': 'end_time', 'artistName': 'artist_name', 'trackName' : 'track_name'})
Spotify’s Web API

In the next section, I use data from Spotify's Web API. I created a developer account here. You do not need a Spotify Premium subscription, but will need a Spotify account to access this tool. The process to retrieve the data is a little complicated. Steven Morse's post titled "Exploring the Spotify API in Python" and John Mannelly's blog post titled "How to Build Your Own Spotify Wrapped with Python, Spotipy and Glide Apps" were excellent resources I used to help me get the information needed.

Part of the reason I chose this route was because the streaming data provided by Spotify (used in the section prior to this) only contained simple information like the artist's name, track name, and date. I was interested in getting more detail about the music I listen to.

I am not going to include the process of retrieving the data here, check out the articles linked above or the project's Jupyter Notebook for details.

Space Missions    

November 2022

  • SQL Queries
  • GitHub Repository
  • Tableau Dashboard

This project explores the Space Missions dataset from Maven Analytic’s Data Playground. This specific dataset was used in the company’s Maven Space Challenge (this is basically a challenge where all participants use the same dataset and use their tool of choice to see who can create the best visualization). I found the data too late to actually participate in this challenge, but did utilize their analysis questions and created a visualization as though I was a participant. I thought this dataset looked interesting and wanted to explore it!

Data


The dataset was downloaded in .csv format and consisted of 4,630 rows of records each representing a space mission from 1957 to August of 2022 and containing information about the name, location, company, rocket, etc.

Import & Preparation

TFor this project, I chose SQL as my tool of choice to clean, transform, and explore the data. To get started, I created a new table in Microsoft SQL Server Management Studio (SSMS) and inserted the data as provided by Maven Analytics

I used simple SQL queries to explore the data and create additional columns that would be used later on while visualizing. I chose to keep all columns (even though not all were utilized) because the dataset only contained 9 columns to begin with. In summary, below are some of the notable transformations performed on the data:

       ⬥ Used the `MissionStatus` column to create dummy columns containing binary variables for each status
       ⬥ Parsed `Location` into individual columns for the country, state, base, and site
       ⬥ New price column which converts previous string format (ex. "1,160" to represent 1,160,000,000) to numeric

Analysis


Below are the questions provided by Maven Analytics for suggested analysis:

       1. How have rocket launches trended across time? Has mission success rate increased?
       2. Which countries have had the most successful space missions? Has it always been that way?
       3. Which rocket has been used for the most space missions? Is it still active?
       4. Are there any patterns you can notice with the launch locations?

For the most part, I took these questions into consideration while performing my analysis but definitely did some additional exploring. 

                                    
--Number of missions attempted per year

SELECT DISTINCT 
    LEFT(date, 4) AS year
    , COUNT(mission) AS 'Count'
FROM projects..spacemissions
GROUP BY LEFT(date, 4)
ORDER BY LEFT(date, 4)

--Success rate (mission_status = 'Success'/ total missions) by year

WITH cte AS (
    SELECT *
        , CASE WHEN missionstatus = 'Success' THEN 1 ELSE 0 END AS status_success
    FROM projects..spacemissions
)
SELECT LEFT(date, 4) AS year
    , ROUND(((CAST(SUM(status_success) AS FLOAT)/CAST(COUNT(status_success) AS FLOAT)) * 100), 2) AS success_rate
FROM cte
GROUP BY LEFT(date, 4)

--Success and failure rates by country

WITH cte AS (
    SELECT *
        , CASE WHEN missionstatus = 'Success' THEN 1 ELSE 0 END AS status_success
        , CASE WHEN missionstatus = 'Failure' THEN 1 ELSE 0 END AS status_fail
        , REVERSE(left(REVERSE(REPLACE((location), ',', '.')), charindex('.', REVERSE(REPLACE((location), ',', '.')))-1)) AS country
    FROM projects..spacemissions
)
SELECT country
    , ROUND(((CAST(SUM(status_success) AS FLOAT)/CAST(COUNT(mission) AS FLOAT)) * 100), 2) AS success_rate
    , ROUND(((CAST(SUM(status_fail) AS FLOAT)/CAST(COUNT(mission) AS FLOAT)) * 100), 2) AS failture_rate
FROM cte
GROUP BY country

--Total number of rockets

SELECT COUNT(DISTINCT(rocket)) AS total_rocket_count
FROM projects..spacemissions

--Number of rockets per country

WITH cte AS (
    SELECT *
        , REVERSE(left(REVERSE(REPLACE((location), ',', '.')), charindex('.', REVERSE(REPLACE((location), ',', '.')))-1)) AS country
    FROM projects..spacemissions
)
SELECT country, COUNT(DISTINCT(rocket)) AS total_rocket_count
FROM cte
GROUP BY country
ORDER BY COUNT(DISTINCT(rocket)) DESC

--Number of countries and launch locations

WITH cte AS (
    SELECT *
        , REVERSE(left(REVERSE(REPLACE((location), ',', '.')), charindex('.', REVERSE(REPLACE((location), ',', '.')))-1)) AS country
    FROM projects..spacemissions
)
SELECT COUNT(DISTINCT(country)) AS country_count
    , COUNT(DISTINCT(location)) AS unique_location
FROM cte

--Total cost

WITH cte AS (
    SELECT *
        , CAST((REPLACE(price, ',', '')) AS float) as price_updated
    FROM projects..spacemissions
)
SELECT SUM(price_updated * 1000000) AS total_cost
FROM cte

--Total cost by country, added success and fail rates to see whether there was a connection between money spent and success

WITH cte AS (
    SELECT *
        , CAST((REPLACE(price, ',', '')) AS float) as price_updated
        , REVERSE(left(REVERSE(REPLACE((location), ',', '.')), charindex('.', REVERSE(REPLACE((location), ',', '.')))-1)) AS [country]
        , CASE WHEN missionstatus = 'Success' THEN 1 ELSE 0 END AS status_success
        , CASE WHEN missionstatus = 'Failure' THEN 1 ELSE 0 END AS status_fail
    FROM projects..spacemissions
)
SELECT country
    , SUM(price_updated * 1000000) AS total_cost
    , ROUND(((CAST(SUM(status_success) AS FLOAT)/CAST(COUNT(mission) AS FLOAT)) * 100), 2) AS success_rate
    , ROUND(((CAST(SUM(status_fail) AS FLOAT)/CAST(COUNT(mission) AS FLOAT)) * 100), 2) AS failture_rate
FROM cte
GROUP BY country
ORDER BY SUM(price_updated * 1000000) DESC

-- Final query used for Tableau file (filled NULLs with 0s)

WITH cte AS (
    SELECT *
        , CASE WHEN missionstatus = 'Success' THEN 1 ELSE 0 END AS status_success
        , CASE WHEN missionstatus = 'Partial Failure' THEN 1 ELSE 0 END AS status_part_fail
        , CASE WHEN missionstatus = 'Prelaunch Failure' THEN 1 ELSE 0 END AS status_pre_fail
        , CASE WHEN missionstatus = 'Failure' THEN 1 ELSE 0 END AS status_fail
        , REVERSE(left(REVERSE(REPLACE((location), ',', '.')), charindex('.', REVERSE(REPLACE((location), ',', '.')))-1)) AS [country]
        , REVERSE(REVERSE(PARSENAME(REPLACE((location), ',', '.'), 2))) AS [state]
        , REVERSE(REVERSE(PARSENAME(REPLACE((location), ',', '.'), 3))) AS [base]
        , REVERSE(REVERSE(PARSENAME(REPLACE((location), ',', '.'), 4))) AS [site]
        , REVERSE(REVERSE(PARSENAME(REPLACE((date), '-', '.'), 3))) AS [year]
        , CAST((REPLACE(price, ',', '')) AS float) as price_updated
    FROM projects..spacemissions
)
SELECT *
    , price_updated * 1000000 as price_final
FROM cte

NOAA Storm Events 2021   

November 2022

  • Jupyter Notebook
  • GitHub Repository

Since being a kid, I have had an interest in storm events. From tornadoes to hurricanes to evensimply rain, I would be watching the Weather Channel waiting for the latest updates. When I discovered that the National Centers for Environmental Information's National Oceanic and Atmospheric Administration (NOAA) published data on all storm events, I knew I had to take a look. This project is focused on exploratory data analysis and simple visualizations in python.

Note: Information on this page is not all of the analysis conducted. Please check out the full repository in GitHub to see everything!

Data


The data published by NOAA is split by year and can be found for download here. To drill down on my area of analysis, I chose to look at storm events that took place in the year 2021 (I would have preferred the latest data, 2022, but the year is not over and I prioritized completeness rather than timeliness). The dataset consists of 3 .csv.gz files (details, fatalities, location) linked by the event ID about storms and other significant weather phenomena:

       ⬥ Details: 51 columns and 61,110 rows
       ⬥ Fatalities: 11 columns and 978 rows
       ⬥ Location: 21 columns and 58,271 rows

At the time of use, all files were last modified 2022-10-18 16:07. More information about the data can be found on the NOAA FAQ page and information regarding the fields/columns can be found here.

Import & Cleaning

The downloaded dataset was provided in .csv.gz format. I would say I am pretty comfortable with .csv files, however I have never encountered a .csv.gz so this required some research on how to handle. I opted to use a Jupyter Notebook for my analysis rather than SQL because I could simply use the .read_csv function from pandas rather than having to convert the file types and mess around with any settings to get them inserted into SSMS.

The first thing I do whenever starting with a new dataset is run a for loop that converts the column names to lowercase and snake case. This makes it easier for future me to reference the columns.



The dates are split up with the year and month in one column and the day in a seperate column (locations does not include the day field). I created a new column that combines these two fields into a complete date.



The damage_property and damage_crops columns contain information about the monetary damage caused by a storm event. The current formatting (ex. $10.00K to represent 10,000), is not useful for aggregate functions. I converted the columns from an object data type to integer and creating a new column called damage_total which is the sum of the two for each row.



In the fatalities dataset, I created bins that categorize the fatality_age into the following categories: '0-18', '19-24', '25-34', '35-44', '45-54', '55-64', and '65+'.'



There was some additional cleaning performed on the datasets, but the above is just a summary of the most notable.

Exploration


To gain a better understaing of the data and how the tables interact with eachother, I performed some exploratory data analysis. This is simple code to see either summary statistics or to get a better understanding of the data characteristics.

Which states had the most storm events? Texas had the most storm events in 2021 with a total of 4,623.



Which storm event types occurred the most? Thunderstorm Wind occurred the most with a total of 15,535.



Of the tornado storm event type, which EF rating occurred most? The EF0 rating occurred the most in 2021, this is not surprising.



In which locations did the most fatalities occur? The top location for fatalities occurred in a permanent home with a total of 168 deaths and vehicle/towed trailer was second with a total of 160.



What percentage was each gender from the total of fatalities? 59.8% of fatalities were male, 28.7% were female, and 11.5% were N/A.



From the binned age categories, which had the top number of fatalities? Ages 65+ contributed to the most fatalities at 29.5%.

Visualization


Since this dataset includes a variety of different data types (like location coordinates), this allowed me the opportunity to create interesting visualizations using some new-to-me packages in python. Screenshotted below are some of my favorite and, in my opinion, most insightful visualizations I created for this project. Again, please check out the corresponding Jupyter Notebook to see all visuals and interact with them if the package allows.

Takeaways



Based on my analysis, hurricanes are the most devastating storm event in terms of both money and lives lost. The highest Enhanced Fujita Scale rating assigned in 2021 was 4 and there were a total of 9 tornadoes that occurred in the year with this rating. Majority of these tornadoes occurred in the same storm event in southwest Kentucky. I was not surprised to see that the top age category for storm event associated fatalities was 65+. However, I was surprised to learn that of this age category, wind accounted for the most deaths. This may be something I research further in the future. My definition of wind may be very different than NOAA’s.

Overall, I really enjoyed this project. It allowed me to practice skills learned in my Masters of Science in Data Science coursework while also leading me to discover new packages and ideologies with python.

United States UFO Sightings    

November 2022

  • Jupyter Notebook

In this project, I explore the UFO Sightings dataset from Maven Analtics’s data playground. Although I may not believe in aliens in the way they are commonly depicted in the media, I still thought this would be an interesting dataset to explore

My process from the uncleaned data to an aesthetcally pleasing visual was to load the dataset into a Jupyter Notebook to clean and explore then transform the pandas DataFrame to a csv to be used in a Tableau dashboard.

Data


As previously stated, the data for this project was downloaded from Maven Analytic’s data playground. The .csv file includes 11 columns and 80,332 rows of UFO sightings across the world from 1909 to 2014. The columns include field such as city, state, country, lat/long, shape, duration, date/time, and comments. Per Maven Analytic’s site, the dat was added on 9/9/2020 and originally sourced from the National UFO Reporting Center.

Import & Cleaning

The first thing I do whenever starting with a new dataset is run a for loop that converts the column names to lowercase and snake case. This makes it easier for future me to reference the columns.



I am going to focus my analysis on UFO sightings that occurred in the United States. Created a new DataFrame called ufo_us which includes all rows where the country is equal to us. This narrowed down the number of rows being analyzed from 80,332 to 65,114.



The ufo_us DataFrame contains rows where the sighting occurred in US territories (ex. Puerto Rico). To narrow down the DataFrame even further, I created a list of the 50 states and am overwiting the ufo_us DataFrame where values in the state column are in the list of states. This narrowed down the number of rows being analyzed from 65,114 to 65,082.



The comments column includes HTML character codes rather than the characters themselves. I replaced the code with their respective characters.



There was some additional cleaning performed on the dataset, but the above is just a summary of the most notable items. Check out the Jupyter Notebook file to see all cleaning.

Exploration


To gain a better understaing of the data, I performed some exploratory data analysis. This is simple code to see either summary statistics or to get a better understanding of the data characteristics. Below are the questions provided by Maven Analytics for suggested analysis:

       1. Figure out which shapes of UFOs tend to be seen most often.
       2. Take a look at data since the year 2000. How have annual sightings trended?
       3. Dig into the US data. Which states report the most UFO sightings? What do you think about this?
       4. Create a histogram to understand the distribution of the duration of UFO sightings (you'll need to do some cleaning first)
       5. Look at the data in the comments column. What other interesting insights can you pull out of this data?

I used these questions as inspiration for my analysis, but did not following them 100%. There were aspects of their questions that I found interested and from there, branched off to my own interests about the data.

What shapes of UFOs tend to be seen most often? The 'light' shape was seen most often at 21.2% of total sightings or 13,471 total times. The 'triangle' shape was seen second most at 10.3% or 6,548 times.



Take a look at data since the year 2000. How have annual sightings trended? Since 2000, the number of sightings by year saw an overall upward trend and peaked in 2012. After the peak, the number of sightings drastically decreased.



Which states report the most UFO sightings? What do you think about this? 'california' saw the most sightings with a total of 8,912 or 13.7% of total sightings. Two factors I believe may play into this are that it is the most populous state and moving east away from the coast, the desert terrain offers clearer skies for sightings.



What time of day did most sightings occur? To better visualize this, I created a new column called time_category which bins the datetime column into the following categories: '0:00 - 3:59', '4:00 - 7:59', '8:00 - 11:59', '12:00 - 15:59', '16:00 - 19:59', '20:00 - 23:59'. There were a total of 31,836 sightings or 48.9% of total sightings during the 8:00 PM - 11:59 PM bin. This may be due to the requirement of darkness to better see anomalies in the sky.



What is the typical duration for a UFO sighting? Created a new column called duration_(minutes) which is simply the duration_(seconds) converted to minutes. Placed these values on a box and whisker plot (excluding outliers). The long upper whisker indicates that there may be a larger variance among the greater values in the fourth quartile. duration_(minutes) is positively skewed. The median duration is around 3 minutes.



Which months have the most sightings? The most sightings were in 'july.' There were a total of 7,649 sightings or 11.8% of the total.

Which words occurred most in the comments? Used .split(), .explode(), and to_list() to create a list of all words in the comments column. Then, used .most_common() to better understand which words were used most. Besides the determiners, some of the most frequent words were 'light(s)', 'over', 'object', 'sky', 'moving', etc.
