If I had to choose a favorite type of data visualization, it would definitely be maps. It is difficult for me to pinpoint the exact reason why these are my favorites, but they also intrigue me most. The information below is not meant to be super insightful, but rather to document my thought process while working to create the final product.

Process


This project started when I stumbled across this image posted by Andrew Gloe on Reddit. I immediately recognized the image's similarity to the cover art for popular album ‚ÄúUnknown Pleasures‚Äù by Joy Division. With a little more research, I learned this type of visual (coined ‚Äújoy plot‚Äù) has come in and out of popularity in the last decade amongst the data community.

Most who create this type of visual do so with R. Recently, I have been trying to strengthen my Tableau skills so I knew that was going to be my tool-of-choice for this type of project. Lucky for me, Konstantin Greger posted an awesome article called "Population Lines ‚Äì the Tableau Edition" which walks you through the process to do so in Tableau. The data they used is from GEOSTAT and is the population count per 1 km grid in Europe. Following Konstantin's tutorial, I was able to create the below visual.



However, this was not enough for me. I wanted to follow the same process but do so with data either about the United States or the world. This led me to Ryan Brideau‚Äôs article on Whack Data called, "Line Graphs and Parallel Processing in R." Ryan used R to create a very similar visual, but the data used encompasses coordinates for the world. This data came from NASA SEDAC (Socioeconomic Data and Applications Center) and was the population by 2.5 arc-minute grid cells.

Sweet this is what I was looking for! The data was provided in a .tif file which is not compatible with Tableau. In the article, Ryan provided the link to his Github repository with all the scripts used. My hope was to use parts of the R script to convert the data to a .csv file with usable formatting which could then be used with Tableau. I was able to get about half way through before RStudio became unresponsive. I am unsure if it was possibly my machine not being strong enough to work through the ~212M rows of data or user error in RStudio, but I deemed myself unsuccessful (even after trying Ryan‚Äôs recommendation of taking full advantage of multi-core CPUs) after spending an afternoon attempting to figure this out. This resource ended up getting scratched.

Next, I found two workbooks on Tableau Public using the same NASA SEDAC data I was hoping to use (World Population Grid by Mike Cisneros and Global Population by Alexander Varlamov). Mike welcomed people to ‚Äúdownload and enjoy‚Äù so I did just that. üôÇ

Making a copy of Mike‚Äôs dashboard to use the data and following Konstantin‚Äôs Tableau tutorial, I was able to create the below visualization:



If I had to summarize what this visualization was composed of in my own words it would be: multiple lines following along the latitude with spikes in the line's height representing population at the latitude‚Äôs corresponding longitude. An aspect of this dashboard I liked that the first dashboard lacked was the availability of coordinates over the seas. This allowed the visual to better replicate the initial Joy Division-esque design. The only major difference I made within the Tableau file (compared to Konstantin‚Äôs process) was the calculated field used to create the line height. Below is the formula I used after messing with the multiplied value until satisfied with the result. 

                                    
[Latitude] + [Population] * 0.0000026
